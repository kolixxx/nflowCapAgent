= Установка netflowAgent на Linux (ubuntu-pc01)
:toc: left
:toclevels: 2
:sectnums:
:revdate: 2026-07-06

Агент на **конечном Linux-хосте**: захват трафика → NetFlow v9 → коллектор `nfcapd`.

Тестовая ОС: **Ubuntu 18.04 x64** (подходит также 20.04 / 22.04 / 24.04).

См. также: link:netflowAgent-install-windows.adoc[Windows], link:netflowAgent-design.adoc[проект], link:logs-cheatsheet.adoc[логи].

== Схема

[source,text]
----
Linux endpoint (ubuntu-pc01)            Коллектор (192.168.30.36)
  libpcap + netflowAgent  ──UDP :9996──►  nfcapd → live/ubuntu-pc01/
                                                 └── nfsen-ng :9000
----

На endpoint *не* ставятся: nfcapd, nfsen-ng, Docker.

[IMPORTANT]
====
Порт **9996** — только для `ubuntu-pc01`. Windows `win-pc01` использует **9995**.
====

== Требования

[cols="1,2", options="header"]
|===
| Параметр | Значение

| ОС
| Ubuntu **18.04** x64 (ваш тест); 20.04+ тоже OK

| Права
| `root` / `sudo` для установки и захвата пакетов

| Сеть
| UDP до коллектора `192.168.30.36:9996`

| Захват
| **libpcap** (`libpcap0.8`)

| Сборка бинарника
| На **той же** машине (или с той же glibc), что и runtime — см. ниже

|===

[WARNING]
====
Бинарник, собранный на Ubuntu 22.04, может **не запуститься** на 18.04 (несовместимость glibc). Собирайте на Ubuntu 18.04 или копируйте исходники и запускайте `build-linux.sh` там.
====

== Быстрый старт (Ubuntu 18.04)

=== 1. Зависимости

[source,bash]
----
sudo apt update
sudo apt install -y libpcap0.8 build-essential pkg-config libpcap-dev curl git
----

=== 2. Rust (если нет cargo)

[source,bash]
----
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y
source "$HOME/.cargo/env"
rustc --version
----

=== 3. Исходники и сборка

[source,bash]
----
git clone https://github.com/kolixxx/nflowCapAgent.git
cd nflowCapAgent/agent
chmod +x scripts/build-linux.sh scripts/install-linux.sh scripts/uninstall-linux.sh
./scripts/build-linux.sh
----

Результат: `agent/dist/linux-x64/netflowAgent` + `config.toml` + скрипты.

=== 4. Config

Отредактируйте `agent/dist/linux-x64/config.toml`:

* `collector.host` — IP коллектора
* `collector.port` = **9996**
* `capture.interface` — `auto` или имя из `--list-devices` (`eth0`, `ens33`, `enp0s3`)

Проверка:

[source,bash]
----
cd dist/linux-x64
sudo ./netflowAgent --list-devices
./netflowAgent --check-config --config config.toml
----

=== 5. Установка службы (systemd)

[source,bash]
----
cd dist/linux-x64
sudo ./install-linux.sh
----

Проверка:

[source,bash]
----
sudo systemctl status netflowAgent
sudo tail -f /var/log/netflowAgent/agent.log
----

Ожидается: `exporting expired flows`, `dest=192.168.30.36:9996`.

=== 6. Удаление службы

[source,bash]
----
cd dist/linux-x64
sudo ./uninstall-linux.sh
----

Файлы в `/opt/netflowAgent`, `/etc/netflowAgent`, логи — остаются.

== Шаг 1. libpcap (подробно)

Аналог **Npcap** на Windows.

[source,bash]
----
sudo apt install -y libpcap0.8
ldconfig -p | grep libpcap
----

Для сборки дополнительно: `libpcap-dev`, `pkg-config`.

== Шаг 2. config.toml (ubuntu-pc01)

[source,toml]
----
[collector]
host = "192.168.30.36"
port = 9996

[export]
format = "netflow9"
observation_domain_id = 1
source_id = 2

[capture]
interface = "auto"
promiscuous = false

[flow]
active_timeout_secs = 60
inactive_timeout_secs = 15

[logging]
level = "info"
file = "/var/log/netflowAgent/agent.log"
----

== Шаг 3. Ручной запуск (отладка)

[source,bash]
----
sudo ./netflowAgent --config config.toml
----

Ctrl+C для остановки. Для захвата нужен root (или `CAP_NET_RAW`).

== Шаг 4. Проверка

=== На endpoint

[source,bash]
----
sudo journalctl -u netflowAgent -f
sudo tail -f /var/log/netflowAgent/agent.log
----

=== На коллекторе

[source,bash]
----
find /var/nfdump/profiles-data/live/ubuntu-pc01 -name "nfcapd.*" | tail -3
nfdump -R /var/nfdump/profiles-data/live/ubuntu-pc01 -c 5 -o extended
nfdump -R /var/nfdump/profiles-data/live/ubuntu-pc01 -o extended 2>/dev/null | grep -c INVALID
----

=== В nfsen-ng

Graphs → source `ubuntu-pc01`. Задержка RRD ~5 мин.

== Устранение неполадок

[cols="1,2", options="header"]
|===
| Симптом | Действие

| `no capture devices found`
| `sudo apt install libpcap0.8`; запуск от root

| `network interface not found`
| `--list-devices`; указать `eth0` / `ens33` в config

| `export failed`
| `ping` коллектора; `ss -ulnp \| grep 9996` на коллекторе; firewall

| Служба `failed`
| `journalctl -u netflowAgent -n 50`; права на `/var/log/netflowAgent`

| `version glibc not found` (бинарник с другой Ubuntu)
| Пересобрать на Ubuntu 18.04: `./scripts/build-linux.sh`

|===
